tooltip方向控制

2018-03-26

问题

今天，在写tooltip的时候总结了一些tooltip的小箭头方向控制，中间或使用到css伪元素::after和content属性来创建一个小箭头标志，在这中间top用于定位箭头，border-width属性指定箭头的大小，border-color用于将内容转换为箭头。

右侧提示框/左侧箭头

dd::after{
    content:"";
    position:absolute;
    top:50%;
    right:100%;
    margin-top:-5px;
    border-width:5px;
    border-style:solid;//边框样式
    border-color:transparent black transparent transparent;//这个箭头颜色设置遵循下，左，上，右的顺序
}

左边提示框/右侧箭头

dd::after{
    content:"";
    position:absolute;
    top:50%;
    left:100%;
    margin-top:-5px;
    border-width:5px;
    border-style:solid;//边框样式
    border-color:transparent transparent transparent black;//这个颜色设置遵循下，左，上，右的顺序
}

顶部提示框/底部箭头

dd::after{
    content:"";
    position:absolute;
    top:100%;
    left:50%;//用来居中
    margin-left:-5px;
    border-width:5px;
    border-style:solid;//边框样式
    border-color:black transparent transparent transparent;//这个颜色设置遵循下，左，上，右的顺序
}

底部提示框/顶部箭头

dd::after{
    content:"";
    position:absolute;
    bottom:100%;
    left:50%;//用来居中
    margin-left:-5px;
    border-width:5px;
    border-style:solid;//边框样式
    border-color:transparent transparent black transparent;//这个颜色设置遵循下，左，上，右的顺序
}

发现

在写的最后加上了，淡入淡出效果，如下

dd:hover .tt{
    visibility:visible;
    opacity:1;
}
.tt{
    opacity:0;
    transition:opacity 1s;
}

但是，当我将visibility更换为display的时候这个淡入效果就失效了，查看了transition属性发现时他不支持display属性的改变，浏览器会将节点属性的变化和display一起显示，导致动画效果失效，解决方法下次有时间更新上来…
