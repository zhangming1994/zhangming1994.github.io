node.js(二)

2018-04-02

Node.js REPL(交互式解释器)

感觉这个其实应该就是说的命令行窗口，node也有并且可以执行读取，打印的功能

使用变量

变量声明使用var关键字，如果没有使用var关键字声明会直接打印出来
使用var关键字声明的变量可以使用console.log()打印出来

多行表达式

支持多行表达式，会在换行的时候自动生成…来进行换行链接的表示

使用(_)下划线来获取上一个表达式的运算结果

Node.js回调函数

Node.js异步编程的直接提现就是回调，异步编程依托于回调来实现，但不能说使用了回调之后程序就异步化了，这是错的。
回调函数在任务完成之后就会被调用，Node所有的API都是支持回调函数的

Node.js事件循环

Node.JS是单进程单线程的应用程序，但是通过事件和回调支持并发，所以性能OK的
Node.js的每一个api都是异步的，并作为一个独立的线程运行，使用异步函数调用，处理并发
Node.js基本上所有的事件机制都是使用观察者模式实现，类似于进入一个事件循环，直到没有观察者就退出，每一个异步事件都生成一个观察者，有事件发生就调用该回调函数

Node.js使用事件驱动模型，当web server接收到骑牛，就把他关闭然后进行处理，然后去服务下一个web请求，当请求完成被放回处理队列，当到达队列开头就被返回给用户
这个webserver是一直接受请求但是不等待任何的读写操作，被称之为非阻塞式io或者时间驱动io

Node.js EventEmitter

Node.js所有的异步io操作在完成的时候都会发送一个事件到事件队列
里面所有的分发事件都是events.EventEmitter的实例

events只提供了一个对象，events.EventEmitter,他的核心就是事件触发和事件监听器功能的封装

EventEmitter的每个事件有一个事件名称和若干参数组成，对于每个事件，EventEmitter支持若干事件监听器

Node.js Buffer(缓冲区）

js语言自身只有字符串数据类型，没有二进制数据类型
但是在处理tcp流或者文件流的时候，必须用到二进制数据，因此定义该Buffer类，用来创建一个专门存放二进制数据的缓冲区
一个Buffer类似于一个整数数组,对应v8堆内存之外的一块原始内存

目前支持如下的字符串编码:

  1. ascii 仅仅支持7位ascii数据
  2. utf8 多字节的unicode编码
  3. utf16le 2或者4个字节
  4. ucs2 utf16le的别名
  5. base64 base64编码
  6. latin1 编成一个字节编码的字符串
  7. binary latin1别名
  8. hex 每个字节编码为两个十六进制的字符

创建Buffer类

  1. Buffer.alloc(size[, fill[, encoding]])： 返回一个指定大小的 Buffer 实例，如果没有设置 fill，则默认填满 0
  2. Buffer.allocUnsafe(size)： 返回一个指定大小的 Buffer 实例，但是它不会被初始化，所以它可能包含敏感的数据
  3. Buffer.allocUnsafeSlow(size)
  4. Buffer.from(array)： 返回一个被 array 的值初始化的新的 Buffer 实例（传入的 array 的元素只能是数字，不然就会自动被 0 覆盖）
  5. Buffer.from(arrayBuffer[, byteOffset[, length]])： 返回一个新建的与给定的 ArrayBuffer 共享同一内存的 Buffer。
  6. Buffer.from(buffer)： 复制传入的 Buffer 实例的数据，并返回一个新的 Buffer 实例
  7. Buffer.from(string[, encoding])： 返回一个被 string 的值初始化的新的 Buffer 实例

写入缓冲区

buf.write(string[, offset[, length]][, encoding]) 返回实际写入的大小
string 写入缓冲区的字符串
offset 缓冲区开始写入的索引值
length 写入的字节数,默认为buffer.length
encoding 使用的编码

从缓冲区读取数据

buf.toString([encoding[, start[, end]]]) 解码缓冲区数据并使用指定的编码返回字符串
encoding 使用的编码
start 开始读取的索引的位置
end 结束位置

将Buffer转换为JSON对象

buf.toJSON(),当字符串化一个Buffer实例的时候,JSON.stringify()会隐式的调用该toJSON()

缓冲区合并

Buffer.concat(list[,totalLangth])
list 用于合并的Buffer对象数组列表
totalLength 指定合并后的Buffer对象的总长度

缓冲区比较

buf.compare(otherBuffer)

缓冲区拷贝

buf.copy(targetBuffer[, targetStart[, sourceStart[, sourceEnd]]])
targetBuffer 要拷贝的对象

缓冲区裁剪

buf.slice([start[,end]])

缓冲区长度

buf.length

Node.js Stream

Stream是一个抽象接口,node.js一共有四种流类型
Readable 可读操作
Writable 可写操作
Duplex 可读可写
Transform 操作被写入数据，然后读出结果
所有的Stream都是EventEmitter的实例
data 当有数据可读的时候触发
end 没有更多的数据可读触发
error 发生错误的时候触发
finish 所有的数据被写入到底层系统的时候触发

操作流中间的数据

  1. 读取 createReadStream()
  3. 写入 createWriteStream()
  4. 管道流 pipe 从输出流到输入流
  5. 链式流 连接输出流到另外一个流并创建多个流操作链的机制

Node.js中间require方法的文件查找策略

node.js存在4类模块,原生模块和3种文件模块
查找顺序:

  1. 是否在文件模块的缓存区中间
  2. 是否原生模块，如果是就看是否在原生模块的缓存中间，是的话直接加载
    没有就加载该原生模块然后缓存
  3. 如果不是原生模块，就在文件模块中间根据扩展名载入文件模块然后缓存该文件模块

Node.js全局对象

在Node.js中间对象是global，所有的全局变量都是global对象的属性，其最根本的作用是作为全局变量的宿主，按照ECMAScript的定义，满足以下条件是全局变量：

  1. 在最外层定义的变量；
  2. 是全局对象的属性；
  3. 隐式定义的变量(未定义直接赋值的变量)；

在Node.js中间是不可能在最外层定义变量的，所有的用户代码都是属于当前模块，二模块不是最外层上下文，永远使用var定义变量避免引入全局变量，因为全局变量会污染命名空间提高代码耦合风险

  1. __filename 表示当前正在执行的脚本的文件名，输出文件所在位置的绝对路径，在模块中间，返回的是模块文件的路径
  2. __dirname 表示当前正在执行脚本所在的目录
  3. setTimeout(cb,ms) 在指定的毫秒数候执行指定函数cb，而且只执行一次，但会一个代表定时器的句柄值
  4. clearTimeout(t) 停止一个之前通过settimeout创建的定时器，t是settimeout返回的句柄值
  5. setInterval(cb,ms) 在指定的毫秒数后执行指定函数cb，返回一个代表定时器的句柄值，可以使用clearInterval(t) 来清除定时器
  6. console 标准输出(.log,.info,.error,.warn,.dir,.time,.timeEnd,.trace,.assert)
  7. process 描述当前Node.JS进程状态的对象(on 退出时触发，)
