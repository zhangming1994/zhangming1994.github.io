Node.js初步

引入requrired模块:使用require指令载入Node.js模块

//使用require指令载入http模块，并将实例化的HTTP赋值给变量
var http=required("http");

创建服务器

//用http.createServer()创建服务器，并使用listen绑定端口，通过request，reponse参数来接收和响应数据
http.createServer(function(request,response)){
    //修改返回头和返回状态码
    response.writeHead(200,{'Content-Type':'text/plain'});
    //发送响应数据
    response.end('Hello World\n');
}).listen(9999);

NPM的使用

NPM是随同Node.JS一起安装的包管理工具,能解决NodeJS代码部署上的很多问题

使用场景

  1. 允许用户从NPM服务器下载别人编写的第三方包到本地使用
  2. 允许用户从NPM服务器下载并安装别人写的命令行程序到本地使用

  3. 允许用户将自己编写的包护着命令行上传到NPM服务器供别人使用
    新版的node.js貌似集成了npm,我在安装完node.js之后就可以直接使用，
    可以升级,命令是:npm install npm -g
    使用淘宝镜像的命令:cnpm install npm -g

    使用Npm命令

    比如安装web框架模块express：npm install express
    全局安装:npm install express -g
    本地安装: npm install express
    查看所有的全局安装模块：npm list -g
    查看某个模块的内容(比如版本号):npm list xxx
    卸载模块: npm uninstall xxx
    查看安装的包: npm ls
    更新模块: npm update xxx（-g 全局的更新）
    搜索模块: npm search xxx
    创建模块: npm init(只是单纯的使用这个，还涉及到在资源库注册用户等内容，还有发布)
    npm资源库注册用户: npm adduser
    发布模块: npm publish
    清空本地缓存: npm cache clear
    帮助:npm help xxx

使用package.json

package.json位于模块的目录下面，(node_modules/xxx/package.json)用于定义包的属性
package.json属性说明

  1. name 包名
  2. version 包版本号
  3. descreption 包描述
  4. homepage 包官网url
  5. author 作者
  6. contributors 包的其他贡献者
  7. keywords 关键字
  8. main 指定了程序的主入口文件，require(‘moduleName’)就会加载这个文件，这个字段的默认值是模块根目录下面的index.js
  9. repository 包代码存放地址的类型 可能是git svn github 等
  10. dependencies 依赖包表 如果依赖包没有安装，npm会自动安装在node_module下面

版本号的问题

NPM使用语义版本号来管理代码
语义版本号氛围XYZ三位，分别代表主版本号，次版本号，补丁版本号，当代码更新的时候，版本号的更新原则如下

  1. 修改bug，更新补丁版本号
  2. 新增加功能，但是向下兼容，更新次版本号
  3. 大的变动，向下不兼容，更新主版本号
